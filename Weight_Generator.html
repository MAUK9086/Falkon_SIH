Detailed Training Process of the Few-Shot Classification Weight Generator

Overview:

The few-shot classification weight generator GGG is designed to infer classification weight vectors for novel categories by using feature vectors from a few training examples. The generator GGG can also optionally utilize the classification weight vectors of base categories WbaseW_{\text{base}}Wbase​, which have been pre-trained on a larger dataset. The training process of GGG aims to effectively transfer knowledge from these base categories to enhance the few-shot classification of novel categories, especially in cases where only one or a few examples are available for the novel class.

  • Learnable Weight Scaling: The final classification weight vector www is a learnable scaled version of the averaged vector:
    w=avg⊙wavgw = \text{avg} \odot w_{\text{avg}}w=avg⊙wavg​

While this approach is simple and effective, it does not fully exploit the knowledge learned from the base categories. Moreover, in the case of one-shot learning (i.e., N=1N=1N=1), averaging cannot accurately infer the classification weight.

3. Training Procedure:

The training process is divided into two stages. During both stages, a cross-entropy loss is minimized to train the feature extractor F(⋅)F(\cdot)F(⋅), the classifier C(W)C(W)C(W), and the weight generator G(⋅)G(\cdot)G(⋅).

Stage 1: Training the ConvNet Recognition Model

  • Goal: To train the feature extractor F(⋅)F(\cdot)F(⋅) and the classification weight vectors WbaseW_{\text{base}}Wbase​ for the base categories.
  • Setup: During this stage, the few-shot classification weight generator GGG is not used. The base categories are treated as a standard classification problem where the ConvNet model learns the feature extractor and the base weight vectors.
  • Loss Function: The cross-entropy loss is used over the base categories:

Stage 2: Training the Few-Shot Classification Weight Generator

  • Goal: To train the weight generator G(⋅)G(\cdot)G(⋅) and refine the base weight vectors WbaseW_{\text{base}}Wbase​.

  • Process: During this stage, the few-shot classification weight generator is trained using "fake" novel categories sampled from the base categories. For each batch, KnovelK_{\text{novel}}Knovel​ novel categories are randomly sampled from the base categories, and the weight generator is used to infer classification weight vectors for these "fake" novel categories.

    Specifically, for each "fake" novel category:

    • NNN examples are sampled.
    • Their feature vectors Z={zi}i=1N\mathbf{Z} = \{z_i\}_{i=1}^{N}Z={zi​}i=1N​ are computed.
    • The few-shot weight generator GGG infers the classification weight vector www using both feature averaging and the attention mechanism.

  • Loss Function: The same cross-entropy loss as in stage 1 is used to train the weight generator. However, the key difference is that the weight vectors for the fake novel categories are generated by G(⋅)G(\cdot)G(⋅) instead of being directly trained.

4. Evaluation and Performance:

During testing, the weight generator G(⋅)G(\cdot)G(⋅) is evaluated on novel categories, which were not seen during training. Few-shot tasks are constructed by sampling KnovelK_{\text{novel}}Knovel​ novel categories, with either one (1-shot) or five (5-shot) examples per category. The model's performance is then evaluated on a separate test set from the same novel categories.

Additionally, the model's ability to retain knowledge about the base categories is also evaluated by assessing its performance on the base category test set.
