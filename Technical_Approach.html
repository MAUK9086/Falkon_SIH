Step-by-Step Breakdown:

1. Audio Preprocessing and Feature Extraction

Librosa for feature extraction:
  • Mel Spectrogram: Represents the short-time power spectrum of the sound on a Mel scale. It is useful for speech and music classification tasks.
  • Spectral Centroid: Indicates where the center of mass of the spectrum is located, which helps distinguish between different types of sounds.
  • Chromagram: Represents the energy distribution across different pitch classes. It's useful for detecting harmony and tonality.
Preprocessing Steps:
  1. Resampling: The audio is resampled to a target sample rate (16 kHz) to ensure consistency.
  2. Padding/Truncating: Each audio clip is padded or truncated to the same length (3 seconds) to maintain uniform input dimensions.
  3. Normalization: Each extracted feature is converted to a log scale (for Mel Spectrogram) or normalized as appropriate (e.g., between 0 and 1 for chromagram).
def extract_audio_features(file_path):
    signal, sr = librosa.load(file_path, sr=TARGET_SAMPLE_RATE)
    
    # Pad or truncate the signal to the target length
    target_length = DURATION * TARGET_SAMPLE_RATE
    if len(signal) > target_length:
        signal = signal[:target_length]
    elif len(signal) < target_length:
        signal = np.pad(signal, (0, target_length - len(signal)))

    # Mel Spectrogram
    mel_spec = librosa.feature.melspectrogram(y=signal, sr=sr, n_mels=N_MELS)
    mel_spec_db = librosa.power_to_db(mel_spec, ref=np.max)

    # Spectral Centroid
    spectral_centroid = librosa.feature.spectral_centroid(y=signal, sr=sr)

    # Chromagram
    chromagram = librosa.feature.chroma_stft(y=signal, sr=sr)
    
    return mel_spec_db, spectral_centroid, chromagram
Stacking Features:

The features are stacked along the depth dimension (i.e., channel-wise) to create a 3D input for the CNN. For each audio sample, the extracted features (Mel, Spectral Centroid, and Chromagram) are concatenated, resulting in 3 channels for the input, similar to RGB channels in images.

def stack_features(mel_spec, spectral_centroid, chromagram):
    mel_spec = mel_spec[..., np.newaxis]
    spectral_centroid = spectral_centroid[..., np.newaxis]
    chromagram = chromagram[..., np.newaxis]

    # Stack along the last axis
    stacked_features = np.concatenate([mel_spec, spectral_centroid, chromagram], axis=-1)
    return stacked_features
Modifying Data Loading Function: 
def load_audio_files(folder_path):
    data = []
    labels = []
    
    for label_folder in os.listdir(folder_path):
        label_path = os.path.join(folder_path, label_folder)
        if os.path.isdir(label_path):
            for file_name in os.listdir(label_path):
                file_path = os.path.join(label_path, file_name)
                
                # Extract Mel, Spectral Centroid, Chromagram
                mel_spec, spectral_centroid, chromagram = extract_audio_features(file_path)
                stacked_features = stack_features(mel_spec, spectral_centroid, chromagram)
                
                data.append(stacked_features)
                labels.append(label_folder)

    return np.array(data), np.array(labels)

2. Transfer Learning with ResNet50

Input Shape:

Now that the input features are multi-channel (Mel, Spectral Centroid, and Chromagram), the input shape for ResNet50 needs to be updated accordingly:

input_shape = (data.shape[1], data.shape[2], 3)  # Height, Width, Channels (3 channels)
Pre-trained Model (ResNet50):

We load a pre-trained ResNet50 model (on ImageNet), using it as a feature extractor. The early layers are frozen to retain the knowledge learned from large-scale image data, and the last few layers are fine-tuned for the specific audio classification task.

resnet = ResNet50(weights='imagenet', include_top=False, input_tensor=Input(shape=input_shape))

# Freeze all layers except the last few
for layer in resnet.layers[:-10]:
    layer.trainable = False
Adding Custom Layers:

We add several fully connected layers after the ResNet50 backbone to adapt the model for multi-class audio classification:

  • GlobalAveragePooling2D: Reduces the dimensionality and flattens the features.
  • Dense: Fully connected layers for classification.
  • BatchNormalization and Dropout: Help improve generalization and reduce overfitting.
  • Output Layer: Softmax activation function for multi-class classification.
x = resnet.output
x = GlobalAveragePooling2D()(x)
x = BatchNormalization()(x)
x = Dropout(0.5)(x)
x = Dense(256, activation='relu')(x)
x = BatchNormalization()(x)
x = Dropout(0.5)(x)
output = Dense(len(np.unique(y_train)), activation='softmax')(x)
Compiling the Model:

The model is compiled with an Adam optimizer, sparse categorical cross-entropy for multi-class classification, and accuracy as the evaluation metric.

model.compile(optimizer='adam', loss='sparse_categorical_crossentropy', metrics=['accuracy'])
Training the Model:

During training, we save the best-performing model based on validation accuracy using ModelCheckpoint.

checkpoint = ModelCheckpoint('best_audio_model.h5', monitor='val_accuracy', save_best_only=True, verbose=1)

Technical Details and Justifications:

  1. Feature Extraction (Librosa):

    • Mel Spectrogram: Captures frequency-based features that are crucial for recognizing speech patterns and tonal information.
    • Spectral Centroid: Helps distinguish different types of sound based on where the "center of mass" of the spectrum is located.
    • Chromagram: Captures harmonic content in audio and is useful for musical and tonal classification.

  2. Preprocessing:

    • Padding/Truncating: Ensures that all audio inputs are of the same length, making the input consistent for the neural network.
    • Stacking Features: We combine multiple feature representations, stacking them along the depth dimension to mimic the structure of multi-channel images (like RGB channels in an image). This allows us to use standard CNNs like ResNet50.

  3. Transfer Learning:

    • ResNet50: Originally trained on ImageNet, ResNet50 can extract complex features from image-like inputs, which can be effectively applied to stacked audio features. Fine-tuning the last few layers of ResNet50 allows us to adapt the model to the specific characteristics of audio data while leveraging pre-trained image features.

  4. Model Architecture:

    • GlobalAveragePooling2D: Reduces the feature map size from the CNN without introducing additional parameters, helping with overfitting.
    • BatchNormalization and Dropout: Both layers are critical to improving the generalization of the model by regularizing the network and preventing overfitting.
    • Dense Layers: Used for classification, the dense layers learn to map the high-level features extracted by ResNet50 to the correct class labels.

Conclusion:

This enhanced approach leverages multiple audio features (Mel Spectrogram, Spectral Centroid, Chromagram) to capture different audio characteristics. By stacking these features and applying transfer learning with ResNet50, we create a powerful audio classification model suitable for various audio tasks. The combination of feature extraction, preprocessing, and transfer learning helps boost model performance, accuracy, and generalization capabilities.
